Title: Why Use Binary

Author: Eric Brauer

Date 2019-01-26

It’s a very good question. We know for a fact that decimal requires fewer digits to represent the same number. How many more? Take a look at this equation:

log210 = 3.322

What this is telling us is that it will take 3.322 more digits to represent a binary number than a decimal number. This means 3.322 more resources, to do the same amount of work. So why do it?

Well, what do we mean when we talk about ones and zeroes anyway? Physically, these are voltage levels. Here’s what keyboard input might look like on an oscilloscope:

The X-axis of an oscilloscope is time. The Y-axis is voltage. In this scenario, there are two voltage levels that the computer understands: high and low.

To re-create a decimal value in this situation, we’d need to recognize ten unique voltage levels, rather than just two. This causes many issues:

  • Voltage levels tend to float. That is, they can become uncalibrated, which means the computer can’t interpret a certain voltage level correctly.
  • Transistors are what make computers possible. Transistors can work like switches, or they can generate AC signals (such as the ones we’d need for decimal levels). However, transistors use far less power when they are either in saturation mode (on), or in cut-off mode.

Power consumption is Voltage * Current and in this graph, as long as either voltage or current is close to zero, the power wasted by transistor will be minimized. Power and heat are the main factors that have limited the speed and size of computers.

  • Digital logic using AND, OR, NOT, XOR gates was already a thing by the time computers came along. Using logic gates to perform math was already an established concept.

Basically, using binary numbers requires more digits (and thus more transistors and memory), but it simplifies the operation of a computer tremendously and so is worth the tradeoff.
